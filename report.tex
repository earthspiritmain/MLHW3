\section{Part 1 - Theoretical aspects}

\subsection*{A -- Physics of the Problem}

\subsubsection{Dimensional Analysis and Scaling}

The problem involves predicting the transversal flow rate field $q(x_i,y_j)\ [\mathrm{m\,s^{-1}}]$ over a $32\times64$ grid from a binary cross-section image and four physical parameters. All quantities with their SI units are:
\begin{itemize}
  \item Pressure drop: $\Delta P\ [\mathrm{Pa}]$
  \item Channel length: $L\ [\mathrm{m}]$
  \item Fluid viscosity: $\mu\ [\mathrm{Pa\cdot s}]$
  \item Pixel area: $\Delta A = \Delta x\,\Delta y\ [\mathrm{m^2}]$
  \item Flow rate field output: $q(x,y)\ [\mathrm{m\,s^{-1}}]$
  \item Total flow: $Q = \sum_{ij}q(x_i,y_j)\,\Delta A\ [\mathrm{m^3\,s^{-1}}]$
\end{itemize}
The binary image $I(x_i,y_j)\in\{0,1\}$ is dimensionless. A unique velocity scale is formed from the four physical parameters by dimensional analysis:
\begin{equation}
  v_0 = \frac{\Delta P\,\Delta A}{\mu\, L}
  \quad\left[\frac{\mathrm{Pa}\cdot\mathrm{m}^2}{\mathrm{Pa\cdot s}\cdot\mathrm{m}}\right]
  = \left[\mathrm{m\,s^{-1}}\right].
  \label{eq:v0}
\end{equation}
This is the only independent velocity scale constructible from $(\Delta P, L, \mu, \Delta A)$; any other dimensional combination is a power of $v_0$. Normalizing $q$ by $v_0$ yields a dimensionless output $\tilde{q}$:
\begin{equation}
  \tilde{q}(x,y) = \frac{q(x,y)}{v_0}
                 = \frac{q(x,y)\,\mu\, L}{\Delta P\,\Delta A},
  \label{eq:dimless}
\end{equation}
so the learning problem reduces to
\begin{equation}
  \tilde{q} = \tilde{F}(I),
  \label{eq:dimless-problem}
\end{equation}
where $\tilde{F}$ maps a dimensionless binary image to a dimensionless field. All four physical parameters collapse into the single prefactor $v_0$. A function $f$ is called a scaling (degree-1 homogeneous) function if
\begin{equation}
  f(\lambda x_1,\dots,\lambda x_n) = \lambda\,f(x_1,\dots,x_n)
  \quad \forall\,\lambda > 0.
  \label{eq:homogeneous-def}
\end{equation}
Since $q$ is linear in $v_0$, the learning problem takes the scaling form
\begin{equation}
  q(x,y;\,\Delta P,L,\mu,\Delta A,I)
  = S\!\left(\frac{\Delta P\,\Delta A}{\mu L},\,I\right),
  \label{eq:scaling-form}
\end{equation}
where $S(\cdot)$ is a scaling function\footnote{$S(\cdot)$ here denotes the degree-1 homogeneous function in the sense of lecture notes Ch.~3. It is distinct from the hydraulic throughput scalar $S_h = \mu L Q/\Delta P\ [\mathrm{m}^4]$ introduced in Ch.~22 of the same notes.} satisfying $S(\lambda v_0,\,I) = \lambda\,S(v_0,\,I)$ for all $\lambda>0$. This encodes the physical law that doubling $v_0$ doubles the flow field everywhere.

\subsubsection{Symmetries of the Problem}

A symmetry of the learning problem is a transformation $T$ of the input image that induces a structured, predictable change in the output. Since $q(x,y)$ is a spatially distributed field, the correct notion is \textbf{equivariance}: a model $f_\theta$ is equivariant under $T$ if
\begin{equation}
  f_\theta(T\cdot I)(x,y) = T\cdot f_\theta(I)(x,y).
  \label{eq:equivariance}
\end{equation}
This contrasts with \textbf{invariance}, which applies to scalar outputs that are unchanged by $T$. The total flow $Q = \sum_{ij}q(x_i,y_j)\Delta A$ is invariant: $Q(T\cdot I) = Q(I)$.

The $32\times64$ rectangular domain admits the following three symmetries:

\begin{enumerate}
  \item \textbf{Left-right (horizontal) reflection} $T_\mathrm{LR}$: flipping $I$ about the vertical midplane yields a horizontally mirrored flow field, $q(T_\mathrm{LR}\cdot I)(x,y) = T_\mathrm{LR}\cdot q(I)(x,y)$.
  \item \textbf{Up-down (vertical) reflection} $T_\mathrm{UD}$: flipping $I$ about the horizontal midplane yields $q(T_\mathrm{UD}\cdot I)(x,y) = T_\mathrm{UD}\cdot q(I)(x,y)$.
  \item \textbf{180° rotation} $R_{180}$: rotating $I$ by $180°$ maps the rectangular domain onto itself, giving $q(R_{180}\cdot I)(x,y) = R_{180}\cdot q(I)(x,y)$.
\end{enumerate}

Note that $90°$ rotations are not symmetries of this problem: a $90°$-rotated $32\times64$ image becomes a $64\times32$ image, which lies outside the input domain. The three transformations above generate the Klein four-group $\mathbb{Z}_2\times\mathbb{Z}_2$ of order 4 ($R_{180} = T_\mathrm{LR}\circ T_\mathrm{UD}$).

\subsubsection{Loss Functions}

Three candidate loss functions are assessed on physical grounds. The Kaggle competition metric is the mean relative error
\begin{equation}
  E = \frac{1}{|\mathcal{D}|}\sum_{i\in\mathcal{D}}
      \frac{1}{N_X N_Y}\sum_{x,y}
      \left|\frac{q_i^T(x,y)-q_i(x,y)}{q_i^T(x,y)+\varepsilon}\right|,
  \label{eq:kaggle-loss}
\end{equation}
which is dimensionless; $\varepsilon$ prevents division by zero at closed pixels. Each pixel contributes equally in relative terms, which is physically meaningful across the wide range of $v_0$ values. The absolute value makes it non-differentiable at zero residual, and $\varepsilon$ introduces an artificial scale below which relative errors are suppressed. The mean squared error
\begin{equation}
  \mathcal{L}_\mathrm{MSE} = \frac{1}{|\mathcal{D}|}\sum_{i}
    \frac{1}{N_X N_Y}\sum_{x,y}
    \bigl[q_i^T(x,y) - q_i(x,y)\bigr]^2
  \label{eq:mse-loss}
\end{equation}
has dimensions $[\mathrm{m^2\,s^{-2}}]$ and is therefore not dimensionless; high-velocity pixels dominate the gradient, and under a global rescaling $q\to\lambda q$ the loss scales as $\lambda^2$, violating degree-1 homogeneity and treating different $v_0$ regimes inconsistently. It is differentiable everywhere and simple to optimize. The dimensionless relative MSE
\begin{equation}
  \mathcal{L}_\mathrm{rel} = \frac{1}{|\mathcal{D}|}\sum_{i}
    \frac{1}{N_X N_Y}\sum_{x,y}
    \left(\frac{q_i^T(x,y) - q_i(x,y)}{q_i^T(x,y)+\varepsilon}\right)^2
  \label{eq:rel-mse-loss}
\end{equation}
is invariant under $q\to\lambda q$ (degree-0 homogeneous) and differentiable everywhere, unlike $E$, which facilitates gradient-based optimization; it assigns larger relative penalties to outliers than $E$ due to squaring. The $\varepsilon$ caveat applies as in $E$.

\subsubsection{Symmetry-Penalizing Loss}

To quantify and penalize violations of equivariance, define the equivariance residual of a network $f_\theta$ on sample $i$ under transformation $T$ as
\begin{equation}
  r_T^{(i)}(x,y) = f_\theta(T\cdot I_i)(x,y) - T\cdot f_\theta(I_i)(x,y).
  \label{eq:equiv-residual}
\end{equation}
A perfectly equivariant network satisfies $r_T^{(i)}\equiv 0$. The symmetry-penalizing loss is
\begin{equation}
  \mathcal{L}_\mathrm{sym} = \sum_{T\in\mathcal{G}}
    \frac{1}{|\mathcal{D}|}\sum_{i}
    \frac{1}{N_X N_Y}\sum_{x,y}
    \bigl[r_T^{(i)}(x,y)\bigr]^2,
  \label{eq:sym-loss}
\end{equation}
where $\mathcal{G} = \{T_\mathrm{LR},\,T_\mathrm{UD},\,R_{180}\}$. This term can be added to any primary loss as a regularizer:
\begin{equation}
  \mathcal{L}_\mathrm{total} = \mathcal{L}_\mathrm{main} + \alpha\,\mathcal{L}_\mathrm{sym},
  \label{eq:total-loss}
\end{equation}
with $\alpha>0$ a hyperparameter. $\mathcal{L}_\mathrm{sym}$ has units $[\mathrm{m^2\,s^{-2}}]$ (consistent with $\mathcal{L}_\mathrm{MSE}$). When the primary loss is the dimensionless Kaggle error $E$, dimensional consistency requires normalizing: replace $\mathcal{L}_\mathrm{sym}$ with $\mathcal{L}_\mathrm{sym}/v_0^2$ so that both terms are dimensionless and $\alpha$ remains a pure number. $\mathcal{L}_\mathrm{sym}$ also serves as a diagnostic metric to compare models trained with and without hardwired symmetries.

\subsection*{B -- Network Design}

\subsubsection{Data Augmentation and Disambiguation}

The symmetry group $\mathcal{G}=\{e,T_\mathrm{LR},T_\mathrm{UD},R_{180}\}\cong\mathbb{Z}_2\times\mathbb{Z}_2$ identified in Section~A provides a free $4\times$ augmentation: for every training pair $(I_i, q_i)$, the three non-trivial transforms generate valid additional samples
\begin{equation}
  (T\cdot I_i,\; T\cdot q_i), \quad T\in\mathcal{G}\setminus\{e\},
  \label{eq:augmentation}
\end{equation}
since the symmetries are equivariances of the ground truth. This inflates $N=1500$ to $N_\mathrm{aug}=6000$ without any additional simulation cost. When $I$ is itself symmetric under some $T$ (e.g.\ left-right symmetric geometry), the augmented pair $(T\cdot I, T\cdot q)$ is identical to the original, so no conflict arises during training. At inference, a model that is not equivariant by design may predict different outputs for $I$ and $T\cdot I$ even when they are the same image. Two strategies address this:
\begin{enumerate}[noitemsep,topsep=0pt]
  \item \textbf{Test-time averaging (TTA):} evaluate on all $|\mathcal{G}|=4$ orbit images and average the back-transformed predictions:
    $\bar{f}(I) = \frac{1}{4}\sum_{T\in\mathcal{G}} T^{-1}\cdot f_\theta(T\cdot I)$.
  \item \textbf{Canonical form:} map each input to a fixed canonical representative of its orbit before the forward pass.
\end{enumerate}
TTA is preferred: it reduces prediction variance for all inputs, not only symmetric ones, and requires no modification of the training pipeline.

\subsubsection{CNN / VGG Design}

A VGG-like architecture maps the binary input $I\in\{0,1\}^{32\times64}$ and the scalar $v_0 = \Delta P\Delta A/(\mu L)$ to the field $q\in\mathbb{R}^{32\times64}$ (Figure~\ref{fig:vgg-arch}). The encoder uses three convolutional blocks each containing two $3\times3$ convolutions followed by $2\times2$ max-pooling, with a fully connected decoder:

\begin{center}
\small
\begin{tabular}{@{}llll@{}}
\toprule
Stage & Layers & Spatial size & Ch.\ \\
\midrule
Input    & --               & $32\times64$ & 1   \\
Block 1  & Conv3$\times$2, Pool2 & $16\times32$ & 32  \\
Block 2  & Conv3$\times$2, Pool2 & $8\times16$  & 64  \\
Block 3  & Conv3$\times$2, Pool2 & $4\times8$   & 128 \\
Bottleneck & Flatten, FC(512), concat $v_0$ & -- & -- \\
Decoder  & FC(2048), reshape & $32\times64$ & 1   \\
\bottomrule
\end{tabular}
\end{center}

The receptive field of each layer is computed using $\mathrm{RF}_\ell = \mathrm{RF}_{\ell-1} + (k-1)J_{\ell-1}$, $J_\ell = J_{\ell-1}s_\ell$, with $\mathrm{RF}_0=1$ and $J_0=1$:

\begin{center}
\small
\begin{tabular}{@{}lrr@{}}
\toprule
Layer & RF & $J$ \\
\midrule

Conv3 (B1) &  5 & 1 \\

After three blocks the receptive field reaches $36\times36$, covering the full $32$-pixel height; the FC decoder covers the full $64$-pixel width. The main limitations are: (i) the FC decoder discards all spatial structure and must reconstruct it from a compressed vector; (ii) parameter count is dominated by the FC layers ($4{\times}8{\times}128{\times}512\approx2{\times}10^6$ for FC(512) and $512{\times}2048\approx10^6$ for the decoder, totalling $\approx3{\times}10^6$ FC parameters); (iii) convolutional layers are translation-equivariant but not $\mathbb{Z}_2\times\mathbb{Z}_2$-equivariant without explicit symmetry hardwiring.

\subsubsection{Fully Convolutional Network}

A fully convolutional (FC-NN) design replaces the flatten and FC decoder with transposed convolutions or bilinear upsampling (Figure~\ref{fig:fcnn-arch}). The scalar $v_0$ is injected as a spatially broadcast feature map at the bottleneck. All layers are spatial, so the network is spatially equivariant throughout and requires far fewer parameters than VGG owing to the absence of large FC layers; output resolution is also adjustable at test time. Without skip connections, however, the bottleneck is the only pathway for spatial information: fine-grained features (e.g.\ sharp zero boundaries at $I=0$) are compressed into a $4\times8$ feature map and must be fully recovered by upsampling alone. Transposed convolutions introduce checkerboard artifacts unless carefully initialized, and performance is typically inferior to U-Net at equivalent parameter counts.

\subsubsection{U-Net}

A U-Net adds skip connections that concatenate encoder feature maps to the corresponding decoder maps at each resolution (Figure~\ref{fig:unet-arch}):
\begin{equation}
  h_\ell^\mathrm{dec} = \mathrm{Conv}\!\bigl([\,h_\ell^\mathrm{enc}\;\Vert\;h_\ell^\mathrm{up}\,]\bigr),
  \label{eq:skip}
\end{equation}
where $\Vert$ denotes channel-wise concatenation and $h_\ell^\mathrm{up}$ is the upsampled feature from the level below. Schematically:
\[
E_1\!\to\!E_2\!\to\!E_3 \;\xrightarrow{\mathrm{bottleneck}}\; D_3\!\leftarrow\!D_2\!\leftarrow\!D_1,
\quad E_\ell\!\xrightarrow{\mathrm{skip}}\!D_\ell.
\]

Skip connections preserve high-frequency spatial detail (including the sharp $q=0$ boundary at closed pixels) without the decoder needing to re-learn it from compressed bottleneck features; the decoder can therefore be shallow, reducing total parameter count. U-Net is the most parameter-efficient architecture for this image-to-image task: at fixed prediction quality it achieves lower parameter count than VGG or FC-NN because skip connections offload reconstruction work from trainable weights to direct feature reuse. The design is however fixed to matching input/output spatial sizes ($32\times64$, satisfied here), and the scalar $v_0$ must be injected and propagated to all decoder levels.

\subsubsection{Hardwiring Symmetries}

Let $\mathcal{G}$ act on inputs by $T\cdot I$ and on outputs by $T\cdot q$. The Reynolds operator converts any function $f_\theta$ into an exactly equivariant one:
\begin{equation}
  \bar{f}(I) = \frac{1}{|\mathcal{G}|}\sum_{T\in\mathcal{G}} T^{-1}\cdot f_\theta(T\cdot I).
  \label{eq:reynolds}
\end{equation}
Equivariance follows immediately: $\bar{f}(S\cdot I)=S\cdot\bar{f}(I)$ for all $S\in\mathcal{G}$, by substituting $T'=TS^{-1}$. For VGG and FC-NN, wrapping either model with \eqref{eq:reynolds} requires four forward passes on the orbit images and averages the back-transformed outputs; the base parameters are unchanged and exact $\mathbb{Z}_2\times\mathbb{Z}_2$ equivariance is enforced at the cost of $4\times$ inference time. For U-Net, \eqref{eq:reynolds} can be applied at the network level in the same way, or alternatively group-equivariant convolutions \cite{cohen2016group} can be used: filters are constrained so that the response to $T\cdot I$ equals $T$ acting on the filter bank of $I$. For our Klein group each learnable filter generates three additional filters $\{T_\mathrm{LR}w, T_\mathrm{UD}w, R_{180}w\}$ by symmetry, reducing the free-parameter count by up to $4\times$ while guaranteeing equivariance throughout every layer, including skip connections.

\subsubsection{Non-Physical Outputs and Prevention}

Two categories of non-physical output can arise:
\begin{enumerate}[noitemsep,topsep=0pt]
  \item \textbf{Flow at closed pixels.} A network may predict $q(x,y)>0$ where $I(x,y)=0$, violating impenetrability. Hard masking — multiplying the raw output by the binary mask:
    \begin{equation}
      \hat{q} = f_\theta(I,v_0)\odot I,
      \label{eq:masking}
    \end{equation}
    is a differentiable, zero-parameter constraint that guarantees $\hat{q}=0$ at all closed pixels.
  \item \textbf{Negative flow values.} Pressure-driven unidirectional flow requires $q\ge0$. Applying $\mathrm{Softplus}(x)=\ln(1+e^x)$ after the mask:
    \begin{equation}
      \hat{q} = \mathrm{Softplus}\!\bigl(f_\theta(I,v_0)\bigr)\odot I
      \label{eq:nonneg}
    \end{equation}
    ensures non-negativity.
\end{enumerate}
Combining both ensures $\hat{q}\ge0$ and $\hat{q}=0$ at $I=0$ by construction, without any additional parameters. Across the three architectures, VGG is least parameter-efficient because the FC decoder dominates parameter count ($\approx3\times10^6$ FC parameters regardless of prediction depth) and convolutional layers are translation-equivariant only; symmetry hardwiring via~\eqref{eq:reynolds} adds no parameters but costs $4\times$ inference time. A fully convolutional NN eliminates the FC layers and is spatially equivariant throughout at substantially lower parameter count, though without skip connections fine spatial detail must still pass through the bottleneck. U-Net with group-equivariant convolutions is the most efficient: skip connections bypass the bottleneck for high-frequency detail allowing a shallow decoder, and group-equivariant convolutions embed exact $\mathcal{G}$-equivariance at every layer with up to $4\times$ fewer free parameters than standard convolutions, at no inference-time overhead.

import numpy as np
    
def incmatrix(genl1,genl2):
    m = len(genl1)
    n = len(genl2)
    M = None #to become the incidence matrix
    VT = np.zeros((n*m,1), int)  #dummy variable
    
    #compute the bitwise xor matrix
    M1 = bitxormatrix(genl1)
    M2 = np.triu(bitxormatrix(genl2),1) 

    for i in range(m-1):
        for j in range(i+1, m):
            [r,c] = np.where(M2 == M1[i,j])
            for k in range(len(r)):
                VT[(i)*n + r[k]] = 1;
                VT[(i)*n + c[k]] = 1;
                VT[(j)*n + r[k]] = 1;
                VT[(j)*n + c[k]] = 1;
                
                if M is None:
                    M = np.copy(VT)
                else:
                    M = np.concatenate((M, VT), 1)
                
                VT = np.zeros((n*m,1), int)
    
    return M
